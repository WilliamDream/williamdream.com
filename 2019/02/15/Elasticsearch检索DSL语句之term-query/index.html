Elasticsearch检索DSL语句之term query

Posted on 2019-02-15

Elasticsearch检索DSL语句之term query

####1. 精确查询

#####1.1 term查询

term过滤器常用来处理数字、布尔值、日期和文本。

GET /library/books/_search
{
  "query": {
      "term": {
        "title": "mysql"
      }
  }
}

通常当查找一个精确值的时候，我们不希望对查询进行评分计算。只希望对文档进行包括或排除的计算，所以我们会使用 constant_score 查询以非评分模式来执行 term 查询并以一作为统一评分。

GET /library/books/_search
{
  "query": {
    "constant_score": {
      "filter": {
        "term": {
          "title": "mysql"
        }
      }
    }
  }
}

term查询会使用小写。

2. 组合过滤器

通常情况下我们查询是复杂的多个条件查询，这时需要使用复合过滤器，可以接受多个其他类型的过滤器作为参数

#####2.1 布尔查询

布尔bool过滤器有三部分组成：

{
   "bool" : {
      "must" :     [],
      "should" :   [],
      "must_not" : [],
   }
}

过滤查询已被弃用,使用 query/bool/must query/bool/filter等进行查询。

GET /library/books/_search
{
   "query" : {
        "bool" : {
          "must": [
            {
              "term" : {"author": "william"}
            }
          ], 
          "should" : [
             { "term" : {"edition" : 3}}, 
             { "term" : {"type" : "java"}} 
          ],
          "must_not" : {
             "term" : {"edition" : 2} 
          }
       }
   }
}

GET /library/books/_search
{
   "query" : {
        "bool" : {
          "should" : [
             { "term" : {"edition" : 3}}, 
             { "term" : {"type" : "java"}} 
          ],
          "must_not" : {
             "term" : {"edition" : 2} 
          },
          "filter": {
              "term" : {
                   "author": "william"
              }
          }

       }
   }
}
2.2 嵌套布尔查询

我们来看看下面的sql查询：

SELECT document FROM books
WHERE  type = "mysql"
OR ( type = "java" AND edition = 4 )

对于这种查询场景需要我们用到嵌套布尔查询，DSL语句如下：

GET /library/books/_search
{
   "query" : {
      "bool" : {
        "should" : [
          { "term" : {"type" : "mysql"}},
          { "bool" : { 
            "must" : [
              {
                "term" : {"type":"java"}
              }, 
              { "term" : {"edition" : 4}} 
            ]
          }}
        ]
     }
   }
}

3. 查询多个精确值

#####3.1 terms查询

想要精确查询多个值可以使用terms查询。

GET /library/books/_search
{
    "query" : {
        "terms": {
          "type": ["spring","java"]
        }
    }
}

4. 范围查询

#####4.1 range范围查询

对于范围过滤我们可以使用Elasticsearch中的range，提供的组合选项有：gt、lt、gte、lte，顾名思义，不解释了。

GET /library/books/_search
{
    "query" : {
      "bool": {
        "must":{
          "range":{
            "publishDate":{
              "gte":"2010-01-01",
              "lte":"2010-12-01"
            }
        }
      }
    }
  }
}

更多日期计算请参考：时间格式参考文档

5. 前缀查询

#####5.1 prdfix query

GET /library/books/_search
{ 
  "query": 
   {
        "prefix" : { "title" : "java" }
  }
}

6. 模糊查询

6.1 wildcard模糊查询
GET /library/books/_search
{ 
  "query": {
    "wildcard" : { "title" : "python*" }
  }
}
